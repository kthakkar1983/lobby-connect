# Foundation Implementation Plan (Plan 1 of 8)

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the empty monorepo shell — pnpm workspaces, Next.js portal scaffold, Vite kiosk scaffold, shared/ui packages, Supabase local dev, plus lint/test/typecheck wired end-to-end. No business features. By the end, `pnpm dev` boots both apps and `pnpm test` passes a trivial smoke test.

**Architecture:** pnpm workspace at the repo root. Two deployable apps (`apps/portal`, `apps/kiosk`) and two internal packages (`packages/shared`, `packages/ui`) that publish types/utilities consumed by the apps via workspace protocol (`workspace:*`). Tailwind v4 in both apps via PostCSS. shadcn primitives live inside `apps/portal/components/ui/` (standard shadcn layout); when kiosk needs reuse, primitives can be promoted to `packages/ui`. Supabase local dev runs via the Supabase CLI; nothing else in this plan touches Supabase. Vercel deployment uses one Vercel project per app, each pointed at its `apps/*` directory.

**Tech stack:**
- pnpm 9+ workspaces, Node 22 LTS
- Next.js 15 (App Router, Turbopack) + React 19 + TypeScript 5
- Vite 6 + React 19 + TypeScript 5
- Tailwind CSS v4 + PostCSS
- shadcn (`shadcn` CLI, "new-york" style)
- Vitest 2 for unit/integration tests
- ESLint 9 (flat config) + Prettier 3
- Supabase CLI for local dev
- Vercel (one project per app)
