## Decisions locked (don't re-litigate)

- **Agent-side only.** Captions show only on the agent's screen. Two-way kiosk captioning (caption the agent onto the kiosk for the guest) was **cut** → documented future seam. This removed a per-property toggle, a migration, kiosk plumbing, and any kiosk consent line.
- **Video ships first, phone second** (both in v1.1). Reason: night call volume skews kiosk (video), and video is the lower-risk integration (Agora hands us the audio track via a known accessor). Phone wiring (Task 7) is purely additive on top of the shared infra.
- **Speechmatics** is the default provider, behind a one-module swap seam (`apps/portal/lib/captions/provider.ts`). Validate accuracy **live** on real calls (Task 8) — no upfront bake-off (there's no stored pilot audio). Swap to AssemblyAI/Deepgram later = a new implementation of that one file.
- **Ephemeral by construction.** No transcript persisted anywhere. **Zero migrations / RLS / new persistence.**
- **Consent:** agent-only + ephemeral is the lowest-risk posture. A quick legal review of real-time-transcription consent law is flagged for **public launch** (not a pilot blocker; not legal advice).

## Verified technical facts (already researched — don't redo)

**Audio track accessors (the guest's remote audio in the agent's browser):**
- Video (Agora): in the existing `user-published` audio handler — `user.audioTrack.getMediaStreamTrack()` (confirmed `IRemoteAudioTrack`, `agora-rtc-sdk-ng@4.24.4`).
- Phone (Twilio): `call.getRemoteStream()?.getAudioTracks()[0]` (confirmed `Call.getRemoteStream(): MediaStream | undefined`, `@twilio/voice-sdk@2.18.3`). Not available synchronously after `accept()` — the plan polls briefly (Task 7d).

**Speechmatics API (source-confirmed 2026-06-25 — sources cited in the plan header):**
- Server JWT mint: `await createSpeechmaticsJWT({ type: "rt", apiKey, ttl })` from `@speechmatics/auth` → JWT string.
- Browser client: `new RealtimeClient(...)`; `await client.start(jwt, { transcription_config, audio_format })`; `client.addEventListener("receiveMessage", ({ data }) => …)`; `client.sendAudio(pcm)`; `client.stopRecognition({ noTimeout: true })`.
- Transcript messages: `data.message` is `"AddPartialTranscript"` (interim) or `"AddTranscript"` (final); text = `data.metadata.transcript`. Partials need `transcription_config.enable_partials: true`.
- US realtime endpoint: `wss://us.rt.speechmatics.com/v2`.
- Audio: `sendAudio` wants PCM chunks; we tap the remote track via Web Audio (AudioContext → ScriptProcessorNode → Int16 PCM). The `@speechmatics/browser-audio-input` helper is mic-oriented and does NOT fit a remote track, so the adapter hand-rolls the tap (with a zero-gain sink so the already-played guest audio isn't echoed).

**Two narrow SDK details to confirm against the installed types in Task 0** (best-effort defaults are already in the plan's code; fallback stated): (a) the `RealtimeClient` constructor option for the US URL (default may be EU); (b) whether `sendAudio` takes an `Int16Array` or needs `.buffer`.

**Test infrastructure:**
- `pnpm -F @lc/portal test` runs BOTH configs: node (`vitest.config.ts`, excludes `tests/components/**`) for lib/route tests, and jsdom (`vitest.jsdom.config.ts`, only `tests/components/**`) for component tests.
- Run one node test: `pnpm -F @lc/portal exec vitest run <path>`. One component test: add `--config vitest.jsdom.config.ts`.
- Route tests mock the supabase clients so `requireApiActor` runs for real (pattern: `tests/app/agora/token.test.ts`). Component tests use `vi.hoisted` + `vi.mock` + `vi.stubGlobal("fetch", …)` (pattern: `tests/components/softphone.test.tsx`).
- `pnpm lint` (repo root) is the ONLY thing that lints `tests/`.

## Guardrails / gotchas

- **No migrations, no RLS, no new persistence** — if you find yourself writing SQL, stop; that's out of scope (and means the two-way kiosk seam crept back in).
- **Captions must never break the call.** They sit on a track we already have, never in the audio path. Any failure → status `error` → the band goes quiet. The token fetch and stream are wrapped so they can't throw into the call UI.
- **Don't double-play guest audio.** The adapter's Web Audio graph uses a **zero-gain sink** — the guest audio is already played by Agora/Twilio. (Watch for echo in the smoke.)
- **Provider stays in one file.** Only `lib/captions/provider.ts` may import `@speechmatics/real-time-client`; only the token route may import `@speechmatics/auth`. If a vendor name appears elsewhere, the seam leaked.
- The plan's code is complete and TDD-ordered. The only intentionally-deferred-to-implementation bits are the two narrow SDK confirmations above.
