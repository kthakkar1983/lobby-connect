## D13 semantics (quick reference)

Live shift = `effectivePresence(status, last_seen_at, now) !== OFFLINE`. Beats (AVAILABLE/AWAY) only REFRESH a live shift — atomic conditional UPDATE, gated → `200 {onDuty:false}` (client flips off), DB error → 204 **fail-open**; ON_CALL bypasses the gate entirely (accepted two-tab edge). `POST /api/presence/go-on-duty` = the only OFFLINE→live transition. `GET /api/presence` = hydration `{onDuty, accepting}` (500 on read error → client fail-open). Client: hydrate-then-beat on mount; off-duty tabs resync read-only on the beat cadence; refresh mid-shift = still on duty (continuity); overnight/swept/lapsed/ended = off duty until the click; laptop-sleep >90s mid-shift wakes OFF duty (named consequence).
