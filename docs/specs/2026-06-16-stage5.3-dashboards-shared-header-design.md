# Stage 5.3 — Dashboards + shared header — DESIGN (LOCKED 2026-06-16)

**Status:** Design locked via `impeccable`; **IMPLEMENTED + merged to `main` 2026-06-16** (agent + admin; browser-verified).
**Scope of this doc:** the shared **gradient header** (all three portals), the **agent dashboard**, and
the **admin command center**. The **owner** dashboard content is a *separate* fresh-chat effort — owner
inherits only the shared header here. **Audio in-call + kiosk** screens are also deferred to later chats.

> **Implementation resolutions (2026-06-16) — where the build deviated from this doc and why:**
> 1. **Agent dashboard is agent-scoped, not pod-wide.** RLS `calls_select` (migration 0004) only lets an
>    agent read calls they personally handled (no assigned-property branch), and this doc barred RLS
>    changes. So the agent view is the agent's own calls, richly broken out by channel/outcome/hour; for
>    the pilot it's near-equivalent (the primary agent handles the calls). True pod-scope = a one-line v2
>    RLS branch. The **admin** view IS genuinely operator-wide (ADMIN reads all operator calls under RLS).
> 2. **Phone-health "needs attention" = recent FAILED calls only.** A property is flagged only on a
>    concrete failure (>= 1 `FAILED` call today). The §3.1 presence-based **coverage-gap** was removed
>    from the impl — it false-alarmed on a *covered* property whose primary agent was simply offline (the
>    normal after-hours setup). Both coverage-gap (needs covered-window + operator-wide admin presence)
>    and the global **"path down"** red (`twilio_webhook` is info-mode, can't tell down from a quiet
>    night) are v2 seams. (Corrected post-merge 2026-06-17 on prod-test feedback.)
> 3. **Softphone stays mounted in the shared layout for BOTH roles** (Kumar's call — the line must persist
>    on every route, as it already did). The §4 "card on home + toast off-home" persistence is honored
>    *without* splitting the component: it's hidden (display:none, still mounted) off-home. The header was
>    also made a **taller, top-aligned** bento so the account dropdown opens into it (not onto a card),
>    and the gradient runs **teal→navy** (left→right).

> Supersedes the brand doc §5.3 "flat and uninspiring" problem statement. This is the answer: real
> depth + hierarchy, channel-aware (Twilio audio vs kiosk/Agora video) operational data, in a bento.

## 0. What changed vs. the current dashboards

- The Stage-2 dashboards were generic stat boxes scoped to *the viewer's own* calls. The locked design
  is **channel-aware** (every `calls` row carries `channel` = `AUDIO`/`VIDEO`) and **data-rich**:
  phone/video split, answered/missed/failed outcomes, live-now, ASA, talk time, avg call length, an
  hourly volume chart, and (admin) a scale-aware phone-health rollup.
- The **persistent 320px right call-rail is removed.** The softphone becomes a **card** inside the
  dashboard bento (agent) / a card on the admin home (admin) — see §4.
- A new **shared gradient header** (navy→teal + static connection-lines) leads every portal.

**Still true (unchanged constraints):** light mode only; brand-semantic colour (mint=live/action,
teal=links, blaze=attention, red=911/destructive only); **no migrations, no new routes, no RLS, no
call/voice/emergency logic changes** — every field already exists. New work = read queries + pure
aggregation helpers (TDD) + composition.

## 1. Shared gradient header (LOCKED — agent, admin, owner)

A single header treatment across all three portals (owner adapts it to its mobile width).

- **Background:** `linear-gradient(112deg, #0E2A45 0%, #13495E 52%, #237E84 100%)` — navy → teal.
  (Earlier experiment mixing in the light `#F4F7F7` tone was **rejected** — reverted to navy→teal.)

  light mint-teal (`#7FE6D9`) at 0.12–0.26 opacity, weights 0.9–1.3, concentrated in the right ~60%
  so the greeting stays clean. Reference path field (SVG `viewBox="0 0 600 80"`,
  `preserveAspectRatio="none"`) is in the locked mockups; reproduce as a static `<svg>` overlay.
- **Greeting** (left): `font-display` (Raleway 600) `Good evening, {firstName}.` in cream `#F4F7F7`.
  **No subtitle** (the old "Covering N properties · Xm talk" line was removed).
- **Account menu** (right): the existing avatar-only "boarding pass" `AccountMenu` (agent/admin).
  **Owner keeps its `UserMenu`.** The avatar wears the `.lc-avatar-halo` ring.
- **Seam hairline:** a 2px `--gradient-seam` (navy→teal→mint) line along the header's **bottom edge**,
  continuous with the navy rail's right-edge seam (rail + header frame the light workspace in an "L").

**Menu-overlap note (settled):** the account dropdown opens down-left from the top-right avatar. With
the rail removed it lands only on the **top-right stat card** (non-interactive) and closes on
click-away — no control is ever covered. No relocation needed.

## 2. Agent dashboard (LOCKED) — pod-scoped, bento

The agent's only route is the dashboard, so the softphone card lives here (stays mounted to ring).

**Layout (top → bottom):**
1. **Header** (§1) — greeting + account menu.
2. **Stat row — 4 cards:** `Answered` · `Missed` (blaze when >0) · `Avg pickup` · **`Avg call length`**.
   Big numbers in `font-mono` (JetBrains Mono), Raleway labels. (Talk-time total moved to the chart
   card, see below.)
3. **Two columns:**
   - **Left (wider):** `Hourly Call Volume` chart (stacked phone/video bars across the shift hours) with
     a small **`Total call duration: X mins`** line under the title; then **`Recent calls`** (channel
     icon + room/lobby · property + answered/missed dot + duration + time).
   - **Right (tall):** the **Softphone card** — `Softphone` label + the **`Line ready`** pill (top-right,
     replacing the old "Ready" dot), the seam-ring idle brand moment (phone icon), "Incoming calls ring
     here", and the **`Accepting calls`** toggle (the agent's AVAILABLE/AWAY presence) at the bottom.
4. **Full-width:** **`Your pod · tonight`** — up to 5 properties, each a stacked **phone/video volume
   bar** (teal=phone, navy=video) + today's count. Reads clean at 1 property and at 5.

**Data sources (all pod-scoped — the agent's covered properties):**

| Panel | Source |
|---|---|
| Stat: Answered | tonight's pod `calls`, `state=COMPLETED` (`answered_at` not null) |
| Stat: Missed | tonight's pod `calls`, `state=NO_ANSWER` |
| Stat: Avg pickup | mean(`answered_at − ring_started_at`) over answered, today (existing `avgPickupSeconds`) |
| Stat: Avg call length | mean(`duration_seconds`) over answered, today **(new helper)** |
| Chart: Hourly Call Volume | tonight's pod `calls` bucketed by hour of `ring_started_at`, split by `channel` **(new helper)** |
| Chart: Total call duration | Σ `duration_seconds` today (existing `sumTodayDurationSeconds`) |
| Recent calls | latest pod `calls` rows: `channel`, room/caller, `state` (answered/missed), `duration_seconds`, time |
| Pod bars | per-property today counts **split by channel** (admin's `startOfTodayUtc` + count-query pattern, ×channel) **(new query)** |
| Line ready / Accepting | softphone phase + `toggleReady` (existing) |

## 3. Admin command center (LOCKED) — operator-wide, bento

**Everything is operator-wide aggregate** (all agents + admins across the operator), not the viewer's
own calls. The two columns **bottom-align** (equal-height columns; trailing card flexes to fill).

**Layout:**
1. **Header** (§1).
2. **Pulse row — 4 tiles:** `Live calls` (mint, "N · X phone · Y video", from `state ∈ {RINGING,
   IN_PROGRESS}` by channel) · `Agents online` (stale-aware presence) · `Open incidents` (blaze) ·
   **`Phone health`** (the scale-aware rollup, see §3.1).
3. **Two columns:**
   - **Left (wider):** a consolidated **Tonight card** = `Hourly Call Volume` (operator-wide,
     "All agents", + Total call duration) **over** an outcomes strip (`Answered` · `Missed` ·
     `Failed` · `Avg pickup` · `Avg call`); then the **Properties board** (per-property primary-agent
     presence + tonight count + the `Covering` toggle; a problem property — e.g. unassigned — is
     flagged **blaze** and floats up).
   - **Right:** the **Softphone card** (admin variant: `Line ready` + seam ring + "You're dialed for
     properties set to Covering" — **no global Accepting toggle**; admin availability is the per-property
     `Covering` toggles on the board); then **`Team on now`** (agents + presence + props covered); then
     an operator-wide **`Recent calls`** feed (fills the bottom-right, balances the columns, scales).

### 3.1 Phone health — the scale-aware rollup (answers "1 of 50 hotels down?")

The single operator `health_signals.twilio_webhook` heartbeat only proves *our whole Twilio path is
up*; it does **not** see per-hotel problems. So the tile is a **rollup**:

- **All good →** mint, `"N / N · lines OK"` (e.g. `5 / 5`).
- **Some hotels failing →** blaze, `"48 / 50 · 2 need attention"`, drill in to the two. Per-property
  "issue" is derived (no per-hotel heartbeat exists): recent `FAILED` calls / coverage gap; the board
  flags those rows blaze too.
- **Whole path down →** red `"phone path down"` only when the global `twilio_webhook` heartbeat goes
  stale.

So one bad hotel out of fifty surfaces as a **count you drill into**, never hidden behind a green light.

**Data sources (operator-wide):** same `calls`/`incidents`/presence aggregations as §2 but scoped to
`operator_id` (not `handled_by_user_id`); `Phone health` rollup = `health_signals.twilio_webhook`
freshness + per-property `FAILED`-call/coverage derivation; `Team on now` = operator AGENT profiles
(widen the existing query to include `id`/`full_name`) + props-covered from `property_assignments`.

## 5. Brand / colour notes for these surfaces

- **Channel colours (categorical, always legended):** **teal `#2EA6AA` = phone/AUDIO**, **navy
  `#0F2D4B` = video/VIDEO**. Both brand anchors, distinct, only ever in charts/bars with a Phone/Video
  legend.
- **Outcomes:** answered = **mint** dot/`live-foreground`; missed (`NO_ANSWER`) = **blaze**; failed =
  **muted/neutral** (system error, less actionable than a missed guest; **not red** — red stays 911).
- **Attention:** open incidents + phone-health issues + unassigned/coverage gaps = **blaze**
  (`attention`). **Live** = mint. **Seam** = line/ring only (header hairline, softphone ring), never a
  fill.
- **Numbers** in JetBrains Mono; **labels** in Raleway (uppercase, tracked); **greeting** Raleway 600.
- **Anti-slop:** no four-identical-gradient KPI cards (the bans still apply — vary weight, semantic
  colour only where it means something); no fabricated data (no CSAT donut, no "vs-yesterday" deltas,
  no live queue/wait — v1 has parallel-dial, no hold queue).

## 6. New pure helpers to TDD (no migrations)

Build test-first under `apps/portal/lib/dashboard/` (and reuse `lib/calls/`):

- `avgCallLengthSeconds(items, now)` — mean `duration_seconds` over today's answered.
- `countByOutcome(items, now)` — `{answered, missed, failed}` by `state`, today.
- `splitByChannel(items)` / per-property `{audio, video}` counts.
- `hourlyVolume(items, tz, now)` — buckets by hour of `ring_started_at`, split by channel (pod for
  agent, operator-wide for admin).
- `countLiveCalls(items)` — `state ∈ {RINGING, IN_PROGRESS}`, split by channel.
- `phoneHealthRollup(properties, calls, heartbeat, now)` — `{ok, total, needAttention[], pathDown}`.
- Reuse: `avgPickupSeconds`, `sumTodayDurationSeconds`, `countOnlineAgents`, `startOfTodayUtc`,
  `countToday`, `isLivePresence`, `isStale`.
