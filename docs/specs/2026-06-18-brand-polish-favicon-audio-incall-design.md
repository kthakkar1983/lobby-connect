## Part 1 — Favicon

**Treatment:** navy tile + reversed mark (brainstorm Option A) — app-icon style, reads on any browser-tab background, matches the navy rail. Chosen over transparent (navy door fades on dark tabs) and white-tile.

### Files (Next.js App Router metadata conventions)

- **`apps/portal/app/icon.svg`** — the primary favicon. A square SVG: navy rounded-square tile (`#0f2d4b`, `rx` ≈ 22% for a soft squircle) with the **reversed mark** (`mark-on-dark` colours: near-white `#f4f7f7` door, teal `#2ea6aa`, mint `#06d6a0`) centered at ~62% of the tile height. Next serves this automatically as `<link rel="icon">`; SVG favicons are universally supported in 2026.
- **`apps/portal/app/apple-icon.png`** — 180×180 PNG for iOS home-screen (owner portal is mobile-responsive and may be saved to a home screen). Apple touch icons must be raster (SVG not supported for `apple-icon`). Generated by rasterizing the same tile at 180×180.

### Authoring details

- The mark's native viewBox is `0 0 770.5 969.7`. Inside a square tile of side `S`, scale the mark to height `0.62·S` and center: `scale = 0.62·S / 969.7`, `tx = (S − 770.5·scale)/2`, `ty = 0.19·S`. Inline the reversed mark paths inside `<g transform="translate(tx,ty) scale(scale)">`.
- No `favicon.ico` fallback (SVG favicon + Next's handling is sufficient).
- The PNG is produced by rendering `icon.svg` (composited onto the navy tile) in a headless browser at 180×180 and saving the screenshot — a one-off build artifact committed to the repo. (Implementation plan picks the exact mechanism; Playwright is already available.)

### Acceptance

- A navy-tile mark appears in the browser tab on every portal page (light and dark tab themes).
- iOS "Add to Home Screen" on the owner portal shows the navy-tile mark.

## Part 2 — Audio in-call overlay redesign

Rework `components/softphone/audio-call-overlay.tsx` to the brand-revision polish bar. **Presentation + two additive features** (local time, Enter-to-save). All existing call/notes/emergency behavior preserved.

### Layout (full-screen overlay, `fixed inset-0 z-50`)

**Header strip** (white, border-bottom):
- Left: pulsing mint live beacon + `On call · {propertyName}`.
- Right: **Call 911** button (red `destructive`), alone in the top-right corner (replaces the old "audio" label). Maximally separated from Hang up. Keeps the existing AlertDialog confirm + `emergencyActive` disabled state ("911 active").

**Emergency / notes banners** — unchanged: `emergencyActive` and `emergencyFailed` banners render below the header exactly as today.

**Body** (`flex-1`) — ≈ **37% call card / 63% playbook** per the approved mockup (this widens today's 25% info rail; "keep the current proportions" referred to the mockup Kumar was viewing, not the old code):
- **Call card** (left, navy `--color-call`), vertical stack:
  1. Small top line: `On call · {mm:ss}` — the call-duration timer, de-emphasized (small mono).
  2. Hotel name `{propertyName}` (semibold), above the animation.
  3. Presence pulse — a calm mint seam-ring animation (the "guest is on the line, audio-only" anchor; honors `prefers-reduced-motion`).
  4. **Hotel local time** — the card's bold anchor: an uppercase mint label `HOTEL LOCAL TIME` over a large bold ticking time (e.g. `11:48 PM`).
  - Removed vs. the old card: the phone-icon circle, the "Guest on the line · audio" subline, and the seam divider.
- **Playbook** (right) — `PlaybookPanel` content unchanged; its `basis` prop set to ~63% (e.g. `basis-3/5`), the card fills the remainder.

**Control bar** (white, border-top):
- Left: **Room #** input + **Notes** input. The notes input carries the save affordance (below).
- Right: **Mute** + **Hang up** (navy `primary`). Removes the leftover `text-[1.1875rem] font-bold` hack → normal button sizing.

### Feature: hotel local time

The property's `timezone` (IANA string, `not null` on every property — `0001_init.sql`) drives a live clock in the call card.

**Wiring (no voice-path / TwiML change):** the softphone already POSTs `/api/twilio/voice/answered` on Accept and ignores the response; the overlay only mounts *after* Accept. So:

- **`app/api/twilio/voice/answered/route.ts`**: on the winning claim, return `200` with `{ timeZone }` (today it returns `204`). Source the timezone from the call's property — extend the operator-scoped read to include it (`fetchOperatorCall` select `"id, state, properties(timezone)"`, or a small follow-up `properties.select("timezone")` keyed by the call's `property_id`). Loser (409) / bad-input paths unchanged.
- **`components/softphone/softphone.tsx`**: in `acceptCall`, read the `answered` response body when `ok`, store `timeZone` in state, pass it to `AudioCallOverlay`. If the POST fails or returns no body, `timeZone` stays null (graceful — see below).
- **`AudioCallOverlay`**: new `timeZone: string | null` prop. Compute the current time with `Intl.DateTimeFormat("en-US", { timeZone, hour: "numeric", minute: "2-digit" })`, recomputed on the shared 1 s tick. Wrap in try/catch (invalid tz → treat as null). When `timeZone` is null/invalid, **hide the local-time block** (no broken/empty state).

**Risk note:** changing `answered` from `204` → `200 {timeZone}` is contract-additive (only the softphone calls it; it ignores the body today). Its route test updates accordingly. The 911 path and all call routing are untouched.

### Feature: press-Enter-to-save (notes)

Reuses the existing save infrastructure (`saveNotes` → `/api/calls/notes` via `reliableFetch`, with the `pendingNotes` durability banner as the post-call backstop).

- **`softphone.tsx`**: `saveNotes` returns `Promise<boolean>` (success) in addition to its current `notesSave`/`pendingNotes` side-effects (additive; existing callers ignore the return). Add `onSaveNotes: () => Promise<boolean>` passed to the overlay, which calls `saveNotes({ callId, roomNumber, notes })` from the refs. Guard: no-op (resolve `true`) when both Room# and Notes are empty.
- **`AudioCallOverlay`**: pressing **Enter** in the Room# or Notes input calls `onSaveNotes()`. The **Notes input carries an in-field ⏎ icon at its right edge** (icon only, no "Enter" text, no separate button — `corner-down-left` from lucide). Local save-indicator state drives the icon:
  - idle → ⏎ (muted)
  - saving → spinner (or pulsing ⏎)
  - saved → mint ✓ for ~1.5 s, then back to ⏎
  - failed → brief red state; the typed text is preserved (it lives in softphone state) and the post-call `pendingNotes` retry banner still fires after the call ends.
- The end-of-call background save (`endCall`) is unchanged — it remains the final flush + durability backstop. Mid-call and end saves hitting the same idempotent endpoint is fine.

### Feature: call-duration timer

Self-tracked in the overlay: capture mount time, `setInterval` every 1 s, format `mm:ss`, clear on unmount. Approximate (mount ≈ answer) and not server-authoritative — acceptable for an at-a-glance duration. Drives both the small duration line and the local-time recompute on the same tick.

### Preserved behavior (must not regress)

- Mute (incl. the live-911 server-side mute branch), Hang up, the 911 emergency conference trigger + confirm dialog, `emergencyActive`/`emergencyFailed` banners, and the notes durability banner — all behave exactly as today. The overlay stays a controlled presentational component; all call/Twilio/emergency logic remains in `softphone.tsx`.
